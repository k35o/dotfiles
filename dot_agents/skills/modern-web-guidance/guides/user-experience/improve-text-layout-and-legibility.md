# テキストレイアウトと可読性を向上させる

特に行末や段落末でのテキストレイアウトは、ページの可読性や視覚的な美しさに影響します。CSS には、特定のユースケースを改善できるテキスト折り返しのオプションがいくつか用意されています。

見出しのような短いテキストブロックには `text-wrap: balance` を使います。このプロパティはブラウザに対して、テキストを行間でできるだけ均等に分配するよう指示し、より対称的な見た目を作り出します。

`text-wrap: pretty` CSS プロパティは、より洗練された折り返しアルゴリズムを有効にすることで本文テキストのタイポグラフィ品質を向上させます。これは特に「オーファン」（段落最終行の単語ひとつだけの状態）を防ぎ、長いテキストブロックでより目に心地よい「ラグ」を作るために設計されています。

## 実装

### 1. **テキスト要素を特定する**:

`text-wrap: balance` には、見出しや表のヘッダーのような短いテキストブロックを選択します。ボーダーや背景といった可視のボックスを持つ要素は避けてください。レイアウト内に予期しない視覚的な空白領域を作ってしまうことがあります。

`text-wrap: pretty` には、孤立した単語（runts）や悪い改行が目立ちやすい長文を含む可能性のある要素を選択します。たとえば次の要素です。
  - `<p>`
  - `<blockquote>`
  - `<li>`
  - その他、長文を含む可能性のある要素

#### 適切な折り返し方法の選択

| 基準 | `text-wrap: balance` | `text-wrap: pretty` | `text-wrap: wrap`（デフォルト） |
| :--- | :--- | :--- | :--- |
| **最適な用途** | 短いブロック（見出し、タイトル） | 長いブロック（段落、リスト） | パフォーマンスがクリティカルなコンテンツ |
| **視覚的なゴール** | 行長を対称にする | オーファン（"runts"）を避ける | 高速で標準的な折り返し |
| **行数の制約** | 最大 6〜10 行（アルゴリズムの上限） | 3 行から多数行まで最適 | 制限なし |
| **パフォーマンスコスト** | **高**: 二分探索アルゴリズム | **中**: 後方探索アルゴリズム | **低**: 標準の貪欲アルゴリズム |

### 2. **選択した折り返しを適用する**:

`text-wrap: balance` は、見出し（`h1`-`h6`）、サブ見出し、プルクオートなど、短い複数行の要素に限定して適用します。

```css
/* Target specific heading elements for balanced wrapping */
h1, h2, h3, h4, h5, h6 {
  /* Enables balanced line-breaking logic */
  text-wrap: balance;
}
```

`text-wrap: pretty` は、段落の最後の数行を評価して最適な改行位置を見つける最適化アルゴリズムを有効にするために使用します。

```css
/* Apply to multi-line text blocks to prevent orphaned words */
p, blockquote, li, .pretty-text {
  /* Enables pretty line-breaking logic for body copy */
  text-wrap: pretty;
}
```

### 重要な制約とパフォーマンス

#### text-wrap: balance

*   **行数制限:** ブラウザは性能維持のためにバランス調整を試みる行数に制限を設けています（典型的には Chromium で **6 行**、Firefox で **10 行**）。これを超えるテキストでは、ブラウザは標準の `wrap` 挙動に戻ります。この制限を超える可能性のあるテキストブロックに `text-wrap: balance` を使わないでください。
*   **限定的な適用:** `text-wrap: balance` をグローバルに適用しないでください（例: `* { text-wrap: balance; }`）。ブラウザが使う反復的な「二分探索」アルゴリズムは計算コストが高いので、特定の短いテキスト要素に限定してください。
*   **幅との関係:** `text-wrap: balance` はコンテナの幅（`inline-size`）を変えません。あくまでその幅 *の中で* テキストがどう折り返されるかに影響します。そのため、コンテナ末尾に空きスペースができる場合があり、フル幅のテキストブロックに依存するレイアウトに影響することがあります。

#### text-wrap: pretty

*   **パフォーマンス vs 品質**: 必須: `text-wrap: pretty` はデフォルトの `wrap`（貪欲）アルゴリズムよりも計算コストが高くなります。複数の行（典型的には最後の 4 行）を評価して改行位置を最適化するためです。ページに極端に大量のテキストがある場合は、すべての要素にグローバルに適用しないでください。
*   **複数行テキストに最適**: `pretty` の効果は 3 行以上の段落でもっとも顕著です。短い 1 行のテキストではほぼ効果がありません。
*   **ブラウザ固有の挙動**: 実装の詳細は異なる点に注意してください。Chromium 系ブラウザは通常、最後の 4 行に注目しますが、他のエンジンは段落全体を評価することがあります。

### フォールバック戦略

text-wrap: balance の Baseline ステータス: Newly available。2024-05-13 から Baseline です。
対応ブラウザ: Chrome 114（2023 年 5 月）、Edge 114（2023 年 6 月）、Firefox 121（2023 年 12 月）、Safari 17.5（2024 年 5 月）。
text-wrap: pretty は限定的に利用可能です。
対応ブラウザ: Chrome 117（2023 年 9 月）、Edge 117（2023 年 9 月）、Safari 26（2025 年 9 月）。
未対応: Firefox。

`text-wrap: balance` や `text-wrap: pretty` がサポートされていないブラウザでは、これらのプロパティは無視され、テキストはデフォルトの `wrap` 挙動で折り返されます。これは標準のタイポグラフィへ優雅にフォールバックするプログレッシブエンハンスメントです。これにより、すべてのブラウザでコンテンツの可読性を保ちつつ、対応ブラウザではより優れた体験を提供できます。

精緻なテキストレイアウトが必須となるクリティカルなレイアウトには、JavaScript ライブラリを使用できますが、低速でパフォーマンス問題を引き起こす可能性がある点に注意してください。
